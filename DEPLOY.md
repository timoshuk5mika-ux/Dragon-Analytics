# Deploy

## Backend: Render

This repo includes `render.yaml` for a Docker-based Render deploy.

1. Create a PostgreSQL database in Neon or Supabase.
2. Copy its connection string.
3. In Render, create a Blueprint from this repository.
4. Set these secret environment variables:

```text
DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require
CORS_ALLOWED_ORIGINS=https://your-frontend-domain.com,http://localhost:5173
```

`JWT_SECRET_KEY` is generated by Render from `render.yaml`.

Render runs the backend Docker image and calls `python -m app.db.init_db` during container startup. That creates or updates tables and seeds `services`.

After Render gives you the backend URL, set the frontend API URL to it:

```text
VITE_API_BASE_URL=https://your-render-service.onrender.com
```

Then rebuild and deploy the frontend.

## Local Docker

```bash
docker compose up --build
```

Backend:

```text
http://127.0.0.1:8000
```

Health check:

```text
http://127.0.0.1:8000/api/v1/health
```

## Database

The backend reads PostgreSQL from `DATABASE_URL`.

For Neon/Supabase, use an SSL-enabled URL, usually with:

```text
?sslmode=require
```
